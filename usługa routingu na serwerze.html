Usługa Routingu w Serwerze

Usługa routingu w serwerze to mechanizm wykorzystywany w aplikacjach webowych, który pozwala na mapowanie przychodzących żądań HTTP na odpowiednie funkcje lub zasoby w aplikacji. Routing dynamiczny jest podstawą wielu nowoczesnych aplikacji, ponieważ umożliwia tworzenie bardziej interaktywnych i elastycznych aplikacji webowych, które dostosowują odpowiedzi w zależności od różnych parametrów, jak np. dane przesyłane przez użytkownika.

Przykłady usług routingu:

  • Frameworki MVC: w takich frameworkach jak Express.js (Node.js) lub Django (Python), routing jest wbudowaną usługą, która mapuje URL do określonych funkcji kontrolera.
  • Routing w aplikacjach SPA: aplikacje jednostronicowe (SPA) używają routingu do dynamicznej zmiany treści bez przeładowania strony, np. React Router w aplikacjach React.

Przykład routingu w Express.js (Node.js):


const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Witaj na stronie głównej!');
});

app.get('/about', (req, res) => {
  res.send('To jest strona o nas!');
});

app.listen(3000, () => {
  console.log('Serwer działa na porcie 3000');
});

W tym przykładzie, routing w Express.js mapuje różne adresy URL (/ i /about) na odpowiednie funkcje zwracające odpowiedzi HTTP.
